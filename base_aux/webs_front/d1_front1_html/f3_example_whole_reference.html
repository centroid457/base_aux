🫧 Параметры bubbles и cancelable при создании событий
При программной генерации события через new Event(type, options) можно задать два важных флага:
bubbles (по умолчанию false) – определяет, будет ли событие всплывать вверх по DOM-дереву (от целевого элемента к родителям, до document).
cancelable (по умолчанию false) – определяет, можно ли отменить действие по умолчанию для этого события с помощью event.preventDefault().
Эти параметры позволяют имитировать поведение настоящих браузерных событий.
📌 Где это полезно?
bubbles: true – если вы хотите, чтобы событие обрабатывалось родительскими элементами (например, делегирование событий).
cancelable: true – если вы хотите дать возможность обработчикам отменить действие, которое должно было произойти (например, предотвратить отправку формы, переход по ссылке).

Родитель
🧠 Пояснение
bubbles: true – событие «поднимется» от кнопки к её родителю, затем к body и т.д. Это позволяет обрабатывать события на уровне контейнера.
bubbles: false – событие дойдёт только до самого элемента (и его обработчиков), родители не узнают о нём.
cancelable: true – внутри обработчика можно вызвать e.preventDefault(), и dispatchEvent вернёт false, показывая, что действие по умолчанию было отменено. Это используется, например, в системах отмены отправки формы.
⚠️ Важно
Для стандартных браузерных событий (например, click, submit) cancelable часто уже равен true. При создании кастомных событий вы сами решаете.
Некоторые события не могут быть cancelable – это зависит от их типа (mousemove, mouseenter и др.).
