типовые

strong — важный текст (жирный + смысловая важность).

b — просто жирный, без дополнительной важности (визуально).

em — акцентирование (курсив + логическое ударение).

i — просто курсив, без дополнительной важности (визуально)

small — мелкий примечательный текст (лицензия, сноска).

sup(E = mc2) — надстрочный

mark — выделенный цветом (как маркер, без выбора цвета)

U+FE0E и U+FE0F Эти специальные символы называются селекторами вариантов (Variation Selectors)
используются для принудительного изменения отображения предыдущего символа
U+FE0E (VS-15): Принудительно переводит emoji в текстовый (монохромный) стиль. Он как бы говорит браузеру: «покажи этот символ как обычный черно-белый значок».
U+FE0F (VS-16): Принудительно переводит символ в emoji-стиль (с цветом). Заставляет браузер отобразить красочную, часто анимированную версию символа.
подходят символы, у которых есть как текстовая, так и emoji-версия, например, универсальный символ предупреждения ⚠️

ATTR WILDCARDS — поиск по значению атрибута не по имени!!!

data-icon="proc_blank" → начинается с "proc" (красный фон)

data-icon="proc_wait" → начинается с "proc" (красный фон)

data-icon="proc_ok" → начинается с "proc" + заканчивается на "ok" (красный фон + подчёркивание)

data-icon="proc_bad" → начинается с "proc" + содержит "bad" (красный фон + зачёркивание)

data-icon="other_value" — ни одно правило не сработало

💡 Как работают селекторы
[data-icon^="proc"] — значение атрибута начинается с "proc" → красный фон.
[data-icon$="ok"] — значение заканчивается на "ok" → подчёркивание.
[data-icon*="bad"] — значение содержит "bad" → зачёркивание.
Все правила применяются одновременно, если условия выполнены.
💡 Иконка вставляется через ::before на основе атрибута. Не нужно добавлять лишние классы или вручную писать символы.

🎯 absolute — привязка к ближайшему позиционированному предку
Блок в потоке

Элемент вырывается из потока и позиционируется относительно ближайшего предка с position: relative/absolute/fixed/sticky.

Если нет позиционированного предка — относительно <body>. Часто используется для выпадающих меню, тултипов.
Всегда ставьте position: relative на родительский контейнер, если внутри используете absolute.
📌 fixed — привязка к окну браузера
fixed HELLO 1212

ИЩИ ВНИЗУ БРАУЗЕРА!!! fixed-элемент привязан к окну браузера. в правом нижнем углу всего окна, а не контейнера.

Обычно используется для «плавающих» кнопок, шапок, баннеров.

Fixed-элемент игнорирует родителя с overflow: hidden и не прокручивается вместе со страницей.
fixed внутри элемента с transform или will-change может привязываться к этому элементу, а не к окну (особенность браузеров).

🎚️ z-index — порядок наложения

z-index: 10

Чем больше z-index, тем выше элемент. Работает только для позиционированных элементов (position не static).

⚠️ z-index работает в контексте родительского stacking context. Не злоупотребляйте большими значениями (используйте 1,2,3…).

TABS

JS-табы

Вкладки (tabs) — это элемент интерфейса, который позволяет переключаться между несколькими панелями с контентом, не перезагружая страницу.

В чистом HTML нет специального тега для табов, но их легко реализовать с помощью CSS и JavaScript (или только CSS с радиокнопками).

✔️ Экономит место на странице
✔️ Улучшает навигацию по группам информации

<div class="tabs-buttons">
  <button class="tab-btn active" data-tab="tab1">Вкладка 1</button>
  <button class="tab-btn" data-tab="tab2">Вкладка 2</button>
</div>
<div class="tabs-content">
  <div class="tab-pane active" id="tab1">...</div>
  <div class="tab-pane" id="tab2">...</div>
</div>
// JS: при клике добавляем/удаляем класс .active

В этом примере используется простой JavaScript: при клике на кнопку активируется соответствующий .tab-pane и меняется стиль кнопки.

  • Важно добавлять атрибуты role="tablist", aria-selected для скринридеров (в учебном примере опущено для краткости, но в реальном проекте используйте).
  • Можно сделать и на чистом CSS: <input type="radio"> + :checked, но JS даёт больше контроля и динамики.
  • Активная вкладка должна быть визуально выделена (цвет, подчёркивание).
  • На мобильных устройствах табы удобно превращать в аккордеон (но это уже продвинутый вариант).
Суть: табы — это не встроенный тег, а популярный паттерн. Здесь показана классическая реализация на JS + CSS.
