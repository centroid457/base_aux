задача для aio-pika
1. обьект работы с сервисом RabbitMq в виде класса,

2. задачи объекта:
- соединение с сервисом (обеспечить стабильное с переподключением),

- добавление консьюмеров (метод для создания извне с параметром имени очереди и функции обработчика),
очереди создаются и подуключаются при запуске метода,
все консьюмеры должны както храниться для последующего корректного завершения при закрытии,

- продьюсеры (метод для отправки данных извне с параметром имени очереди и тела данных),
очереди создаются и подуключаются при запуске метода с одним именем,
продьюсеры не отпрявляют данные напрямую, а аккумулируют например в dequeue (для каждого имени очереди отдельная либо одна очередь для всех - выбери сам лучший вариант),
сообщения отправляются из хранилица по методу FIFO,

- корректное завершение всех фоновых задач (консьюмеров/продакшенов/соединения/и всех остальных порожденных обьектом),
